# My Simple Paint App

### 🔗 [웹사이트 방문하기](https://nagyun-oh.github.io/my-first-project/)

<br>

## 해당 프로젝트를 한 목적
1. **강의 학습 정리**:HTML + CSS + JavaScript 강의 학습 정리
2. **협업 도구 숙련**: Git과 GitHub를 활용한 버전 관리 및 배포 프로세스 익히기
3. **배포 과정 숙련** : 로컬 환경에서 개발한 파일을 실제 서버(GitHub Pages)에 올리는 전체 워크플로우 경험

<br> 

##  주요 기능
1. **자유로운 드로잉**: 마우스를 이용해 선을 그릴 수 있습니다.
2. **색상 & 두께 조절**: 원하는 색과 굵기를 선택하세요.
3. **지우개 모드**: `destination-out` 기술을 사용한 정밀 지우개.
4. **이미지 저장**: 내가 그린 그림을 파일로 저장(Download)합니다.
5. **되돌리기/다시하기** : 그림을 되돌리거나 다시하기 할 수 있습니다.

<br>

##  사용 기술
* **Language**: JavaScript (ES6+)
* **Graphic**: HTML5 Canvas API
* **Style**: CSS3 (Flexbox, Dark Mode UI)

## 배운 점
* **실시간 에러 핸들링** : "가장 좋은 에러는 컴파일 에러다"라는 말의 의미를 체감했습니다. 자바스크립트는 런타임에 에러가 발견되는 경우가 많아, 브라우저 콘솔을 통한 디버깅과 꼼꼼한 코드 작성의 중요성을 배웠습니다.
* **사용자 경험(UX)**: 지우개 모드 시 커서 모양을 변경하거나, 저장 시 배경을 흰색으로 처리하는 등 사용자의 편의를 고려한 디테일의 중요성을 느꼈습니다.
* 웹 서버는 기본적으로 index.html을 대문으로 인식한다는 것을 배웠습니다. 파일명 하나로 페이지 연결 여부가 결정되는 것을 보며 웹 표준 명칭의 중요성을 알게 되었습니다.
